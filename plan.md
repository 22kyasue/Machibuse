# Machibuse（待ち伏せ） - 開発計画

## Tech Stack（技術スタック）

| Layer | Technology |
|-------|-----------|
| Frontend | Next.js 14 (App Router) + TypeScript |
| Styling | Tailwind CSS |
| Backend/API | Next.js API Routes (Route Handlers) |
| Database | Supabase (PostgreSQL) |
| Auth | Supabase Auth |
| Notifications | Supabase Realtime + Email (Resend) |
| Scraping (後日) | Python scripts (V1以降) |
| Deployment | Vercel |

# 注意事項

- **全ての出力・会話・コメント・UIテキスト・データは日本語で行うこと（英語は一切使用しない）**
- データベース設計は「物件（アセット）」中心であり、「広告（リスティング）」中心ではない
- ステータスラベルは全画面で一貫して表示すること
- 閲覧設計の原則: 募集一覧を見せるのではなく、建物と間取りタイプの「在庫理解」をさせる
- 各画面は「情報表示」ではなく「意思決定」ページとして設計する
- 賃料より先に「文脈」を見せる（建物 → タイプ → 状態 → 動き → 賃料 の順）
- 初期はエリアを絞る（港区・渋谷区・千代田区の高級賃貸/タワマン）
- 写真サイトではなくデータサイトとして設計（表、タグ、履歴、状態、比較重視）
